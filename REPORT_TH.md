# รายงานสรุปโปรแกรม YOLO Custom Object Detection

## 1) บทคัดย่อ
โครงการนี้เป็นระบบตรวจจับวัตถุแบบกำหนดคลาสเอง (Custom Object Detection) โดยใช้ Ultralytics YOLO สำหรับงานฝึกโมเดล (training), ทดสอบการตรวจจับ (inference) และครอปวัตถุที่ตรวจพบออกมาเป็นภาพย่อยอัตโนมัติ โปรแกรมถูกออกแบบให้ใช้งานได้ทั้งแบบคำสั่ง (CLI) และแบบหน้าต่างกราฟิก (GUI) สำหรับงานครอปภาพ

---

## 2) วัตถุประสงค์
- พัฒนา pipeline สำหรับเทรนโมเดลตรวจจับวัตถุจากข้อมูลที่ผู้ใช้เตรียมเอง
- รองรับการแบ่งชุดข้อมูล train/val/test อัตโนมัติ
- รองรับการทดสอบกับภาพเดี่ยว, โฟลเดอร์ภาพ, วิดีโอ และเว็บแคม
- เพิ่มเครื่องมือครอปวัตถุจากผลตรวจจับเพื่อนำไปใช้งานต่อ

---

## 3) โครงสร้างโปรแกรมและหน้าที่ไฟล์หลัก
- `prepare_data.py` : เตรียมโครงสร้าง dataset และแบ่งข้อมูล train/val/test
- `train.py` : ตรวจสอบ dataset, โหลดโมเดลตั้งต้น และเทรนตามค่าใน `config.yaml`
- `test.py` : ทำ inference และบันทึกผลตรวจจับ
- `crop.py` : โปรแกรม GUI สำหรับเลือกโมเดล/ภาพ แล้วตรวจจับและบันทึกภาพครอป
- `config.yaml` : ไฟล์กำหนดคลาส, พาธข้อมูล และ hyperparameters
- `requirements.txt` : รายการไลบรารีที่ต้องใช้
- `setup.sh` และ `setup.bat` : สคริปต์ตั้งค่าสภาพแวดล้อมเบื้องต้น

---

## 4) ขั้นตอนการทำงานของระบบ
1. **เตรียมสภาพแวดล้อม**
   - สร้าง virtual environment และติดตั้ง dependency จาก `requirements.txt`
2. **เตรียมข้อมูล**
   - เก็บภาพที่ `raw_data/images`
   - เก็บ label แบบ YOLO ที่ `raw_data/labels`
   - รัน `prepare_data.py` เพื่อคัดลอกและแบ่งข้อมูลไปที่ `dataset/images/*` และ `dataset/labels/*`
3. **ฝึกโมเดล**
   - รัน `train.py` โดยอ่านค่าจาก `config.yaml`
   - โปรแกรมจะตรวจสอบจำนวนภาพ train/val ก่อนเริ่มฝึก
4. **ทดสอบโมเดล**
   - รัน `test.py` โดยกำหนดโมเดลและ source ที่ต้องการทดสอบ
5. **ครอปวัตถุที่ตรวจพบ**
   - ใช้ `crop.py` เพื่อเลือกรูปภาพและบันทึก crop ตาม bounding box

---

## 5) การตั้งค่าที่สำคัญ (จาก `config.yaml`)
- คลาสที่ใช้งานปัจจุบัน: `0: dogNose`
- โมเดลตั้งต้น: `yolov8n.pt`
- จำนวนรอบฝึก: `epochs: 100`
- ขนาดภาพอินพุต: `imgsz: 640`
- ขนาดแบตช์: `batch: 16`
- ตัวเพิ่มประสิทธิภาพ: `optimizer: SGD`

---

## 6) ผลการฝึกโมเดล (อ้างอิงจาก `runs/detect/runs/train/yolo_custom/results.csv`)
ค่าผลลัพธ์ช่วงท้ายการฝึก (Epoch 100):
- Precision (B): **0.99824**
- Recall (B): **1.00000**
- mAP@0.5 (B): **0.99500**
- mAP@0.5:0.95 (B): **0.91335**

สรุป: โมเดลมีความแม่นยำสูงมากสำหรับคลาสที่ฝึก และมีความสามารถโดยรวมดีในเกณฑ์ mAP@0.5:0.95

## 8) จุดเด่นของโปรแกรม
- โครงสร้างแยกหน้าที่ชัดเจน (เตรียมข้อมูล/เทรน/ทดสอบ/ครอป)
- ใช้ไฟล์ config กลาง (`config.yaml`) ปรับการฝึกได้สะดวก
- รองรับการใช้งานหลายรูปแบบ (CLI + GUI)
- มี artifact การฝึกครบสำหรับวิเคราะห์ผล

---

## 9) ข้อจำกัดและข้อเสนอแนะ
- ปัจจุบันเป็นงานคลาสเดียว (`dogNose`) หากต้องการหลายคลาสต้องเพิ่มใน `names`
- ควรตรวจสอบความสมดุลข้อมูลและความหลากหลายฉากเพื่อเพิ่มความทนทานของโมเดล
- ควรทดสอบ inference บนข้อมูลที่ไม่เคยเห็นจริงเพิ่มเติม เพื่อประเมินการใช้งานภาคสนาม

---

## 10) สรุป
โปรแกรมนี้เป็นระบบ YOLO ที่พร้อมใช้งานจริงตั้งแต่เตรียมข้อมูลจนถึงนำโมเดลไปใช้งานตรวจจับและครอปวัตถุ ผลการฝึกล่าสุดแสดงประสิทธิภาพสูงมากสำหรับโจทย์ที่กำหนด และมีไฟล์ผลลัพธ์เชิงวิเคราะห์ครบถ้วน เหมาะสำหรับต่อยอดใช้งานเชิงผลิตภัณฑ์หรือทำรายงานเชิงวิชาการ
